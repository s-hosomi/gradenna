# gradenna — differentiable FDTD antenna inverse design (web visualizer)

A static Vite + TypeScript + Three.js visualization site for the gradenna project.
Deployable to GitHub Pages.

## Prerequisites

- Node.js >= 18
- npm >= 9
- (Optional) [wasm-pack](https://rustwasm.github.io/wasm-pack/) for the Live FDTD view
- (Optional) Python environment with gradenna installed for data generation

## Development

```bash
cd web/app
npm install
npm run dev
```

The dev server starts at http://localhost:5173. Without data files or the wasm kernel
each view shows a fallback message — the site remains fully functional.

## Build

```bash
npm run build
```

Output goes to `web/app/dist/`. The build succeeds even without the wasm kernel or
data JSON files (dynamic imports and fetches are wrapped in try/catch).

## Type check

```bash
npm run typecheck
```

## Generating visualization data

Run the export script from the project root:

```bash
JAX_ENABLE_X64=1 .venv/bin/python scripts/export_viz.py
```

This writes JSON files to `web/app/public/data/`:

- `optimization.json` — topology optimization frame sequence
- `farfield3d.json` — 3-D far-field directivity pattern
- `s11.json` — S11 return-loss curve (gradenna + optional openEMS reference)

## Building the wasm kernel (Live FDTD view)

```bash
cd web/wasm-kernel
wasm-pack build --target web --out-dir pkg
```

After building, the Live FDTD view becomes active automatically when you run
`npm run dev` or reload the built site. The wasm binary is bundled by Vite at
build time if `pkg/` exists.

## Deploying to GitHub Pages

### Manual (gh-pages branch)

```bash
cd web/app
npm run build
npx gh-pages -d dist
```

### GitHub Actions (example workflow)

Create `.github/workflows/pages.yml` in the repo root:

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
    paths: ["web/app/**"]
permissions:
  contents: read
  pages: write
  id-token: write
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - name: Install and build
        run: |
          cd web/app
          npm ci
          npm run build
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: web/app/dist
      - uses: actions/deploy-pages@v4
```

Enable Pages in the repository settings (source: GitHub Actions) and push.

## Project structure

```
web/app/
  index.html           entry HTML
  vite.config.ts       Vite config (base:"./", @wasm alias, fs.allow)
  tsconfig.json        TypeScript config
  src/
    main.ts            Tab routing and app shell
    types.ts           JSON data types and wasm module interface
    colormaps.ts       Cold / viridis / diverging colormap utilities
    views/
      OptimizationView.ts   Density map + convergence chart (Three.js + canvas 2D)
      LiveFdtdView.ts       Real-time 2-D FDTD (wasm) with conductor painting
      FarField3DView.ts     3-D far-field sphere mesh with OrbitControls
      S11View.ts            S11 return-loss plot (canvas 2D)
  public/
    data/              JSON data files (generated by scripts/export_viz.py)
      .gitkeep
```
